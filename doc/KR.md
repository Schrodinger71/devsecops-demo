## 3. Глава 1. Теоретические основы DevSecOps

### 1.1 Понятие DevOps и DevSecOps

**Ключевые моменты:**

* DevOps — объединение разработки и эксплуатации
* DevSecOps — добавление безопасности «by design»
* Shift Left Security

**Можно написать:**

> DevSecOps представляет собой развитие концепции DevOps, в рамках которого вопросы информационной безопасности интегрируются в процесс разработки с самых ранних этапов. В отличие от традиционного подхода, где безопасность рассматривалась как завершающий этап, DevSecOps предполагает непрерывный контроль и автоматизацию защитных механизмов.

---

### 1.2 Жизненный цикл DevSecOps

Опиши этапы:

1. Планирование
2. Разработка
3. Сборка (Build)
4. Тестирование
5. Развертывание
6. Эксплуатация и мониторинг

## 5. Глава 3. Анализ уязвимостей приложения

### 3.1 Потенциальные уязвимости

На основе **реального кода** (очень важно для курсовой):

#### 1️⃣ Хардкод секретов

```python
API_KEY = os.getenv("API_KEY", "dev-key")
```

**Риск:**

* Использование слабого значения по умолчанию
* Возможная компрометация

#### 2️⃣ Отсутствие HTTPS

* Передача API-ключа в открытом виде
* Возможность MITM-атаки

#### 3️⃣ Недостаточная аутентификация

* API-ключ вместо JWT/OAuth
* Нет разграничения ролей

#### 4️⃣ Логирование пользовательских данных

```python
logger.info(f"User requested: {username}")
```

**Риск:**

* Утечка персональных данных в логах

---

### 3.2 Анализ угроз (STRIDE — можно кратко)

| Угроза                 | Описание              |
| ---------------------- | --------------------- |
| Spoofing               | Подмена API-ключа     |
| Information Disclosure | Утечка логов          |
| Denial of Service      | Отсутствие rate-limit |
